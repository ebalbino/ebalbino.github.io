Why I enjoy the terminal

Posted on Jan 23, 2026

If there’s one thing I enjoy about being a software engineer, it’s the typing.

Everything else frustrates me occasionally. This doesn’t mean I don’t enjoy those aspects of the job. Debugging errors piques my curiosity. Designing systems with my peers keeps me sharp. Building software is fun. But I also know how much sleep those shenanigans have cost me.

Except for typing. Typing always goes smoothly. It’s the silver lining that threads everything together. As long as I’m typing, it feels like I’m moving toward something.

Considering my knack for typing, it should come as no surprise that I live in the terminal. But I’m no purist. I don’t do everything through a shell. I launch my games from Steam. I browse the web with Firefox/Chromium.

But I’m usually booting up my computer to write something. During my day, I will write:

  • Code
  • Tests
  • Documentation
  • Prompts
  • Email
  • Meeting notes
  • One off notes
  • Lists
  • Essays
  • Scripts
  • Outlines

I enjoy typing. While many find the terminal to be a cumbersome interface since one is constrained to the keyboard. This exact feature is what sparks joy in me.

Day to day, I’m switching between modes of writing. Since it’s all in the terminal, switching between these modes becomes another mode of writing. By always typing, I’m never breaking my flow.

It’s a simple one-two. I type text. The computer responds. Rinse and repeat. The loop continues until I’m finished for the day.

Something interesting about this loop is that it’s the same mental loop I use when programming. Thus when I’m using my terminal, I’m not only typing all the time. I’m always programming as well. Two birds, one stone.
